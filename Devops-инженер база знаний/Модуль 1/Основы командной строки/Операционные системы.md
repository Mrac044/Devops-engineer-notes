## Появление командной строки
Появилась в 1970-х, тогда компьютерами называли **мейнфреймы** - большие машины, занимающие целые этажи зданий, в то же время начинали появляться Unix-система, которая позволяла подключаться к мейнфрейму через **терминал** - устройства, которые нужны были для ввода и вывода информации
## Стандарт POSIX
**Стандарт POSIX** - Portable Operating System Interface for UNIX, сейчас это целое семейство стандартов, описывающих работы ключевых компонентов операционной системы и способов взаимодействия с ними из программ написанных на си
==возник из-за ситуации, когда на основе юникса создавались новые системы, программы на которых были не совместимы с другими системами, и разработчики для каждой новой системы писали новую программу.==
## Операционная система Linux
==В 1991 году студент Линус Торвальдс решил написать собственную операционную систему, которая совместима со стандартом POSIX. Благодаря интернету, Линус собрал целое сообщество разработчиков, которые довольно быстро создали рабочую операционную систему под названием **Linux** и выложили ее исходный код в открытый доступ. С тех пор популярность Linux для серверного окружения быстро росла.==

Linux - лишь ядро операционной системы, которое не получится использовать без сопутствующих программ

Когда Линус начал свою разработку, уже существовал проект GNU, в рамках которого были разработаны свободные программы как основной инструмент разработчика программ на языке Си:
- Текстовый редактор **Emacs**
- Компилятор языка Си **gcc**
- Командная оболочка **bash**
- Библиотека важнейших функций для программ на Си **libc**

Сейчас Linux не используется в чистом виде. Вместо этого конечные пользователи имеют дело с **дистрибутивами**. Дистрибутив включает в себя:
- Сам Linux
- GNU-программы
- Утилиты для установки и настройки Linux
- Поддержку регулярного обновления самого ядра и его окружения

В отличие от Unix-подобных операционных систем, Windows изначально создавалась для однопользовательского режима и никогда не пыталась быть совместимой с POSIX. Ей удалось захватить рынок персональных компьютеров, но на серверах она почти не встречается, несмотря на существование Windows Server.

Windows редко ставят на серверы, потому что она платная, и совсем не совместима с POSIX. При этом у Windows есть командная строка `cmd.exe`, но ее функциональность отличается от командной строки Linux, в ней используются другие команды. Опыт работы с ней не сильно поможет при работе в Linux-окружении, с которым все равно придется столкнуться.

Если вы пользуетесь macOS, то можно ничего не делать — эта операционная система основана на Unix и считается даже более POSIX-совместимой, чем Linux. У нее прекрасная командная строка и инструментарий для работы.
