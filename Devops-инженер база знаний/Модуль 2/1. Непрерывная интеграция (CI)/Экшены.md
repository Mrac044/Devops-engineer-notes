В предыдущих уроках мы уже встречались с несколькими экшенами. #Скрипт Чаще всего в сборках используется экшен _checkout_, который клонирует репозиторий в рабочую директорию:

```
steps:
  - uses: actions/checkout@v4
```

- Экшен работает как один из шагов задания.

- Для этого вместо ключа `run` используется ключ `uses`

- Имя экшена берётся из [каталога экшенов](https://github.com/marketplace?type=actions)(могут быть как встроенные в гитхаб, так и созданные пользователями). Имена соотвествуют структуре ссылок самого гитхаба: *имя пользователя или команды/название репозитория* (встроенные экшены находятся в команде *actions*)

- Для использования в воркфлоу требуется указать ещё и версию экшена, чтобы версия случайно не обновилась и случайно не поломались все воркфлоу, которые с экшеном работают. Следить за версиями нужно самому, поглядывая в ридми конкретного репозитория с экшеном

- #Скрипт У экшена могут быть параметры. Они задаются через ключ `with`:

```
steps:
  - uses: actions/checkout@v4
  # https://github.com/actions/setup-node
  - uses: actions/setup-node@v4
    with:
      node-version: '18.x'
      cache: 'npm' # ускоряет повторные сборки
  - run: npm ci
  - run: npm test
```

#Скрипт А вот пример [стороннего экшена](https://github.com/marketplace/actions/cypress-io), который запускает тесты на фреймворке cypress:

```
name: End-to-end tests
on: [push]
jobs:
  cypress-run:
    runs-on: ubuntu-22.04
    steps:
      - uses: actions/checkout@v4
      # Устанавливает зависимости, кеширует их и запускает тесты
      - uses: cypress-io/github-action@v6
```
