**GitHub Actions** — бесплатная для публичных репозиториев система непрерывной интеграции.

Функции Github Actions:
- Запуск проверки кода линтером и тестами
- Деплой проекта
- Опубликовка новой версии пакета
- Подключение оповещения в мессенджер о событиях в репозитории и др...

Принцип работы Github Actions: в нужном репозитории создается директория _.github/workflows_, внутри которой размещаются файлы с описанием шагов, которые нужно выполнить на различные действия (файлы yaml)

#Скрипт Ниже описание процесса "show-directory", который запускается на пуш в репозиторий.
```
# file: .github/workflows/show-directory.yml
name: show-directory
# on – определяет события, которые запускают воркфлоу
on: push
jobs:
  # build – произвольно выбранное имя задания
  # их может быть больше одного
  build:
    # операционная система для работы воркфлоу
    runs-on: ubuntu-latest
    steps: # список шагов, которые надо выполнить
      # экшен — выполняет какую-то задачу
      # checkout — клонирует репозиторий
      - uses: actions/checkout@v4
      # run – произвольная bash-команда
      # ls -la выведет содержимое текущего репозитория
      - run: ls -la
```
**Workflow** - файл в Github Actions с описанием какого-либо процесса с расширением yaml.

Основные концепции Github Actions: 
![[Pasted image 20260112225232.png]]
- **Воркфлоу / Workflows** - может быть несколько в репозитории, каждый определяется в отдельном файле в каталоге _.github/workflows_, могут выполняется несколько параллельно
- **События / Events** - события в репозитории, после которых запускаются ворквфлоу (пуш, пул-реквест, запуск воркфлоу в определенное время(cron), запуск от вызова  Webhook API Github)
- **Задания / Jobs** - из них состоит воркфлоу, содержит набор команд, которые запускаются вместе с рабочим процессом, по умолчанию выполняются параллельно в одном воркфлоу, но можно настроить зависимость, чтобы они выполнялись последовательно
- **Раннеры / Runners** - на них выполняются задания, временный сервер на гитхабе с выбраннос ОС. **Автономные раннеры** - позволяют создать свое окружение для выполнения экшена
- **Шаги / Steps** - из них состоят задания, это либо команда оболочки, либо экшен. Шаги выполняются последовательно на раннере, связанным с заданием (по умолчанию в случае сбоя шага все следующие пропускаются)
- **Экшен / Actions** - многократно используемый блок кода, который может служить шагом задания (как функции в питоне), принимает параметры и создает значения, которые потом можно использовать в других экшенах. Экшены доступны на: [GitHub Marketplace](https://github.com/marketplace?type=actions).
