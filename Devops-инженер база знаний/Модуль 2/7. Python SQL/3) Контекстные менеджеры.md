Для обработки таких случаев можно использовать стандартный синтаксис try-except (если выполнение sql может привести к ошибке)

```python
sql = "INSERT INTO users (username, phone) VALUES ('tommy', '123456789');"
cursor = conn.cursor()
try:
    cursor.execute(sql)
except:
    print("Cannot execute SQL-query")
finally:
    cursor.close()
```

#Реальная_работа Теперь, если возникнет любая ошибка, то курсор правильно закроется. В теории это выглядит нормально, но на практике, об этом сложно постоянно помнить и программисты регулярно забывают добавлять блок `finally`. Да и весь синтаксис выглядит громоздким. Потому создатели библиотеки добавили синтаксис контекстных менеджеров для объектов соединения и курсора:

```python
conn = psycopg2.connect("postgresql://user:password@host:port/hexlet_test")
sql = "INSERT INTO users (username, phone) VALUES ('tommy', '123456789');"
with conn.cursor() as curs:
    curs.execute(sql)

conn.close()  # закрываем соединение
```

Важное замечание, что несмотря на то, что привычные вам контекстные менеджеры освобождают ресурсы после выхода из блока `with`, здесь соединение не закрывается. И потому важно закрыть его принудительно самим.
