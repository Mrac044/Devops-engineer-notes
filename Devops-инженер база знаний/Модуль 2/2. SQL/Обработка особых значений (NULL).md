==Дело в том, что проверка значения `NULL` с помощью `=`, `>`, `<` и других операторов сравнения всегда будет давать результат `NULL`.==
## Как работать с `NULL`

#SQL-команда Для сравнения и проверки на `NULL` необходимо использовать операторы `IS` и `IS NOT`, а не стандартные операторы сравнения Правильный запрос будет выглядеть так:

```SQL
SELECT * FROM users WHERE birthday IS NOT NULL;
```

[View on DB Fiddle](https://www.db-fiddle.com/f/637d1tmrRhxktNHWghMwNY/1)

**`IS` и `IS NOT`** - операторы, которые используются для определения `NULL`-значений и различных состояний

Бывает полезно проверить, во всех ли строках обязательные поля заполнены.
#SQL-команда Выведем строки с незаполненным полем `created_at`:

```SQL
SELECT * FROM users WHERE created_at IS NULL;
```

[View on DB Fiddle](https://www.db-fiddle.com/f/637d1tmrRhxktNHWghMwNY/2)

Если такой запрос вернул какие-то строки, это может говорить о сбое в системе.

При этом важно знать, что `NULL` и строка нулевой длины — это не одно и то же. В отличие от строки нулевой длины, `NULL` говорит, что никакое значение не занесено в текущую ячейку базы данных.

Еще одна интересная особенность — `NULL` не равен `NULL`. Так происходит, потому что нельзя сравнить одну пустоту с другой. Следующий запрос вернет пустую таблицу:

```SQL
SELECT * FROM users WHERE NULL = NULL;
```
