**HTTP** – текстовый протокол, с помощью которого взаимодействуют клиент, например, браузер и сервер.

Пользователь шлет определенный запрос на сервер, запрашивая или передавая нужные данные, а сервер, в зависимости от запроса, выполняет нужную логику и возвращает результат, обычно это HTML-страница либо редирект.

Для того чтобы посмотреть, как работает HTTP, мы будем использовать сервис [HTTP Server](https://http.hexlet.app/http-protocol-openapi), созданный специально для экспериментов.

Для запросов используется специальная утилита, которая называется _telnet_ ([пример HTTP-запроса, выполненного с помощью утилиты telnet](https://asciinema.org/a/10703)).

## Что такое telnet?

**Telnet** — это консольная программа, с помощью которой можно вручную установить соединение с удалённым сервером по сети и напрямую обмениваться данными.

Telnet устанавливает TCP-соединение с указанным IP-адресом и портом, после чего всё, что вы вводите, отправляется серверу напрямую.

Например, так можно подключиться к нашему учебному серверу, эту команду нужно набрать в терминале:
#Telnet
```shell
# Передаем адрес сайта и указываем TCP-порт
# Происходит подключение к серверу по протоколу TCP
telnet http.hexlet.app 80
```

### HTTP и нижележащие протоколы

Однако сам по себе HTTP не может соединять удаленные компьютеры; для этого используются другие протоколы, такие как TCP.
TCP создает канал для общения между программами на удаленных компьютерах, для чего необходимы IP-адрес целевого компьютера и порт, на котором работает нужная программа.

```
HTTP (запросы и ответы)
   ↓
TCP (надёжная доставка, порты)
   ↓
IP (адресация между компьютерами)
   ↓
Ethernet / Wi-Fi (физическая передача данных)
```
Команда telnet выше делает именно это, она выполняет соединение по TCP и только после этого входит в режим взаимодействия по HTTP.

И на этом моменте возникает два вопроса:

1. Мы передали адрес сайта, откуда берется ip-адрес? Все сетевые программы, среди которых браузеры и telnet, выполняют преобразование имени сайта в его ip-адрес.  #Telnet Делается это с помощью системы DNS, еще одного столпа интернета.

```shell
# Пример того, как можно узнать ip-адрес с помощью утилиты ping
# В вашем случае адрес может быть другим, ip-адреса могут меняться
# Опция -с устанавливает количество отправляемых пакетов
ping -c 4 http.hexlet.app # 188.114.97.0

# Затем можно использовать его для соединения с сервером
telnet 188.114.97.0 80
```

2. Почему порт имеет номер 80? 2. Это общепринятое соглашение. Сайты, доступные по HTTP, доступны на порту 80, а по HTTPS – на порту 443. Именно поэтому в браузерах порты не указываются, браузер подставляет их автоматически.

#Telnet Если соединение произошло успешно, то telnet выводит на экран такие строчки:

```shell
telnet http.hexlet.app 80

Trying 188.114.97.0...
Connected to http.hexlet.app.
Escape character is '^]'.
```

## Что из себя представляет сам запрос?

![[Pasted image 20260222222729.png]]

Запрос состоит из нескольких частей. Первая часть — _стартовая строка_ (_request line_). Вторая — заголовки.

1. В _стартовой строке_ мы указываем специальное слово, еще говорят "глагол", они определяют, как реагировать на этот запрос. Глагол **HEAD**, он очень простой, и просит сервер отдать только заголовки, без содержимого. Более распространенным является GET. Именно с помощью GET мы запрашиваем содержимое сайта.
2. После глагола указывается путь к ресурсу _request URI_. Если мы указываем `/`, это обозначает просто корень сайта. Ресурс может быть любым, например, `/users` — может быть ресурсом пользователей. Имя ресурса зависит не только от того, к какому ресурсу мы хотим получить доступ, но и от сервера. Разные серверы предоставляют доступ к разным ресурсам.
3. Дальше все, что нужно сделать, это указать название протокола и его версию. В принципе этого достаточно, и для 1.0 больше ничего делать не нужно:
#HTTP-запрос
```http
HEAD /http-protocol/example HTTP/1.0
```

4. Дальше идут заголовки. Заголовки позволяют передавать дополнительную информацию. Количество стандартных заголовков достаточно большое, помимо них можно добавлять любые свои. 

Мы указываем имя и через двоеточие какое-то значение: _REFERER: value_. Заголовки часто указывают заглавными буквами, но регистр здесь не важен.

Браузерами используется много заголовков, например, _user-agent_. Этот заголовок используется для аналитики, а также, когда необходимо адаптировать страницы сайта под разные экраны или браузеры. Но и без него все должно работать:

```http
HEAD /http-protocol/example HTTP/1.0
User-Agent: google сhrome
```

>HTTP — текстовый протокол. Все правила основаны на простых соглашениях. 
>Например, несколько заголовков отделяются друг от друга переводом строки (и никак иначе!). 
>
>А каким образом сервер поймет, что вы закончили передавать данные?
> 
> В HTTP это делается с помощью двух переводов строки. После этого сервер считает что все данные были отправлены и больше данных не будет. То есть фактически два перевода строки (перевод после последнего заголовка и пустая строка) приводят к отправке данных.

В ответ на запрос приходит **response**, который состоит из **status line**, например, `HTTP/1.1 200 OK` (В HTTP определено множество статусов, таких как 400 и 500, которые информируют о различных ситуациях).

В конце мы видим одну интересную деталь: _Connection closed by foreign host._ Запрос соединения был закрыт внешним хостом. Так работает практически все в интернете. Обычно серверы настроены на 30-секундный интервал и закрывают соединение, если в течение этого интервала ничего не приходит.
